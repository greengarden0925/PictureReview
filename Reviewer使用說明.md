# PictureReview 使用說明（審查者與管理員）

本文件說明：**誰需要安裝 Docker**、**審查者（Reviewer）要如何開啟與使用網頁**，以及**管理員部署前環境該準備什麼**。  
技術架構與 API 細節見 [project design.md](project%20design.md)；指令速查見 [README.md](README.md)。

---

## 一、誰需要安裝 Docker？

| 角色 | 是否需要 Docker | 說明 |
|------|-----------------|------|
| **管理員**（負責架站的人） | **需要** | 在主機或自己的電腦上執行 `docker compose`，讓大家用「網址」連線。 |
| **審查者（Reviewer）** | **不需要** | 只要瀏覽器能開管理員提供的網址即可。 |

### 管理員：Docker 要裝哪一種？

- **Windows、macOS**  
  請安裝官方 **[Docker Desktop](https://www.docker.com/products/docker-desktop/)**（安裝後需啟動，托盤圖示顯示為運行中，再執行 `docker compose`）。  
  本專案測試環境常見為：**Windows 10/11** + **Docker Desktop** + **WSL2**（安裝 Docker Desktop 時可依精靈啟用）。

- **Linux 伺服器（例如 Ubuntu）**  
  一般安裝 **Docker Engine** 與 **Docker Compose 外掛**（`docker compose` 指令），**不必**安裝 Docker Desktop。

- **審查者電腦**  
  **不必**安裝 Docker、不必安裝 Node.js（除非改走 README 的「本機雙擊」備選方案）。

## 二、審查者（Reviewer）環境與操作

### 2.1 您需要準備什麼？

1. **可上網的電腦或平板**（建議以電腦為主，螢幕較大較適合看圖）。
2. **瀏覽器**（建議使用最新版的 **Chrome**、**Edge** 或 **Firefox**）。
3. **網路**  
   - 若網址是內部主機（例如 `http://192.168.x.x:8080`），請先連上 **院所／學校 VPN** 或 **院內網路**（依單位規定）。  
   - 若管理員提供的是 **https://** 公開網址，則在一般網路下即可開啟（仍建議使用 HTTPS）。
4. **管理員提供給您的三項資訊**  
   - **網址**（例如 `https://review.example.com` 或 `http://主機名稱:8080`）  
   - **網站登入帳號**（HTTP Basic Auth 的使用者名稱）  
   - **網站登入密碼**（**明文密碼**，是您平常輸入的那一組字，**不是**很長的 `$2a$10$...` 雜湊字串）

### 2.2 第一次使用：步驟說明

1. **開啟瀏覽器**，在網址列輸入管理員給您的 **完整網址**，按 Enter。  
2. 若瀏覽器跳出 **「登入」或「需要驗證」** 的小視窗：  
   - **使用者名稱**：輸入管理員提供的帳號。  
   - **密碼**：輸入管理員提供的**明文密碼**。  
   - 按 **登入**／**確定**。  
3. 通過後會進入 PictureReview 網站。依頁面指示輸入 **審查者名稱**（用於標註這份審查是誰填的，建議使用本名、工號或約定好的代號，且**同一專案內盡量固定**）。  
4. 進入 **審查頁** 後，您會看到圖片與問卷：  
   - 依序瀏覽每一組圖片。  
   - 填寫問卷（量表、開放題等）。  
   - 使用 **上一頁／下一頁** 切換組別；若有 **放大預覽**，可點圖查看細節。  
5. 填寫過程中請依頁面提示 **儲存**（若有），避免關閉分頁後未寫入。

### 2.3 兩種「帳密」不要搞混

| 名稱 | 用途 | 誰設定 |
|------|------|--------|
| **網站登入（瀏覽器跳出小視窗）** | 防止未授權的人開啟網站 | 管理員（寫在伺服器設定裡） |
| **審查者名稱（網站內輸入）** | 審查紀錄上要顯示您是誰 | 您自己輸入（建議與單位約定格式） |

**問卷後台／管理設定** 另有 **管理用 Token**（若啟用），僅給負責改題目的人，審查者通常**不需要**。

### 2.4 常見問題排除

| 現象 | 可能原因 | 建議作法 |
|------|----------|----------|
| 無法連上網址、一直轉圈 | 未連 VPN、網址錯誤、主機未開機 | 確認網路與 VPN；聯絡管理員確認服務是否啟動。 |
| 跳出登入視窗但一直失敗 | 帳密打錯；誤把雜湊當密碼 | 密碼應為**一般文字密碼**；向管理員重新確認帳密。 |
| 畫面顯示 401、Unauthorized | 帳密錯誤或伺服器設定問題 | 聯絡管理員檢查帳密檔與 Nginx 設定。 |
| 沒有圖、列表是空的 | 伺服器上尚未放入圖檔資料 | 聯絡管理員確認已同步 `output` 資料。 |
| 只能 http、無法 https | 測試環境常見 | 在可信網路下使用；正式對外建議管理員改為 **HTTPS**。 |

## 三、管理員：用 Docker 架站前的環境檢查

以下由 **負責部署的人** 執行，審查者可略過。

### 3.1 軟體需求

1. **Docker**  
   - Windows／Mac：**Docker Desktop** 已安裝且**正在執行**。  
   - Linux：**Docker Engine** + `docker compose`。

2. **專案與設定**  
   - 已取得 **PictureReview** 專案資料夾（含 `docker-compose.yml`、`Dockerfile`）。  
   - 在專案根目錄可執行終端機指令。

3. **（建議）Node.js 18+**  
   - 僅用於在本機產生 **`.htpasswd`**（`npm run write-htpasswd`）與選用的 `npm run docker:check`。  
   - **執行中的容器不依賴您電腦上的 Node**，映像內已含執行環境。

### 3.2 部署前必做事项（摘要）

1. **產生 Nginx 帳密檔**（UTF-8，勿手動用記事本亂存成 UTF-16）：  
   `npm run write-htpasswd -- 帳號 明文密碼`

2. **準備圖檔目錄**  
   - 在 `PictureReview/docker-data/output/` 放入與父專案 **`output/`** 相同內容（整份複製）。

3. **（建議）設定管理 Token**  
   - 在 `PictureReview` 同層建立或編輯 **`.env`**，內容包含：  
     `PICTURE_REVIEW_ADMIN_TOKEN=您的長隨機密鑰`  
   - 再執行 `docker compose up -d --build`，避免他人任意改問卷。

4. **啟動**  
   ```bash
   cd PictureReview
   docker compose up -d --build
   ```  
   預設對外 **8080**（可改環境變數 `HTTP_PORT`）。

5. **給審查者**  
   - 對外網址（含埠號或網域）  
   - Nginx **明文**帳號與密碼  
   - 若為內網，說明是否需 **VPN**

6. **正式環境**  
   - Basic Auth 在 **HTTP** 下傳輸風險較高，建議前面加 **HTTPS**（反向代理或雲端 Load Balancer）。

### 3.3 自我檢查指令

```bash
cd PictureReview
npm run docker:check
docker compose ps
```

詳細除錯見 [README.md](README.md) 的「Docker 除錯與確認可否執行」。
